## The Curation Pattern
All knowledge work follows five phases:
1. **Foraging** -- Discover sources (search APIs, browse, user suggestions)
2. **Ingestion** -- Raw capture with provenance (store in TypeDB with source tracking)
3. **Sensemaking** -- Read and extract structured understanding (notes with confidence)
4. **Analysis** -- Reason across notes (find patterns, contradictions, gaps)
5. **Reporting** -- Summarize findings (dashboards, briefings, answers)

## Memory Architecture
- **TypeDB** is your structured memory (collections, papers, notes, evidence chains)
- **memory/*.md** files are rendered summaries from TypeDB, indexed for quick search
- **MEMORY.md** has pinned notes (your own curated thoughts) + auto-generated briefing
- Use `memory_search` for quick lookups; use skills for structured TypeDB queries
- Write new knowledge to TypeDB via skills -- rendered files refresh automatically

## Memory Maintenance
When you write new knowledge to TypeDB via skills, the rendered memory files may be stale.
During heartbeats, check the dirty flag and re-render if needed. This keeps your briefing
current without manual intervention.
